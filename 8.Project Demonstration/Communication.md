## 2. Key Talking Points for Viva & Evaluation Presentation

### 1. Highlighting Architectural Ingenuity (`store.js` vs `Mongoose`)
* **Communicate:** Explain how you did not just write standard Mongoose routes. Explain the dual-engine `global.isMongoConnected` check.
* **Why it matters:** Demonstrates deep mastery of Node's file system (`fs`), error handling, and robust systems design beyond basic CRUD.

### 2. Emphasizing User Delight & Real-Time Tracking (`UserHome.jsx`)
* **Communicate:** Point to the 4-stage interactive dispatch tracker and the driver profile card (`Vikram Sharma - 4.9 ★ - DL 01 AB 1234`).
* **Why it matters:** Proves your ability to create dynamic stateful UI updates that respond to external backend changes.

### 3. Showcasing Full-Stack Scope Ownership
* **Communicate:** Note that as a solo developer, every line of React styling (`index.css`), authentication interceptors (`authMiddleware.js`), schema modeling (`Booking.js`), and QA testing was individually authored.
* **Why it matters:** Validates complete technical independence and high productivity.
